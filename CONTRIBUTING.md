# Contributing to React Agency Portfolio

First off, thank you for considering contributing to React Agency Portfolio! It's people like you who make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Check if the bug is already reported** by searching the [issues](https://github.com/aadhar41/React-Agency-Portfolio/issues).
- **Use the Bug Report template** if available.
- **Provide as much detail as possible.** Include steps to reproduce, what you expected to see, and what happened instead.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

- **Check if the enhancement has already been suggested.**
- **Use the Feature Request template** if available.
- **Describe the current behavior and the behavior you'd like to see.**
- **Explain why this enhancement would be useful.**

### Pull Requests

Standard process for submitting changes:

1. **Fork the repo** and create your branch from `master`.
2. **Install dependencies** using `npm install`.
3. **Make your changes**.
4. **Follow the code style** of the project.
5. **Add tests** if applicable.
6. **Submit a Pull Request** with a clear description of what you've done.

## Style Guide

- Use descriptive variable names.
- Keep components small and focused.
- Use functional components and hooks.
- Comment complex logic.

Thank you for contributing!
